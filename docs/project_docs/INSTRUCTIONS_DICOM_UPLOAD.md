# 📋 ИНСТРУКЦИЯ: ЗАГРУЗКА DICOM ФАЙЛОВ НА СЕРВЕР

## 🏥 ТЕКУЩАЯ СТРУКТУРА DICOM

Ваши DICOM файлы организованы следующим образом:
```
D:\DICOM СНИМКИ\
├── Мовсесян Ярика 08.09.2024 Натив/
│   ├── 24090213/           # Серия исследования
│   │   ├── 00000001        # DICOM slice (527KB)
│   │   ├── 00000002        # DICOM slice (527KB)
│   │   └── ... (всего ~707 слайсов)
│   ├── DICOMDIR           # Индекс DICOM
│   └── [служебные файлы]
└── [другие пациенты...]
```

## 🚨 ПРОБЛЕМЫ В ТЕКУЩЕЙ РЕАЛИЗАЦИИ

### 1. **Неправильная обработка структуры**
- **Код ожидает:** ZIP → [DICOM файлы в корне]
- **Реальность:** ZIP → Пациент/Дата/Серия/[DICOM файлы]

### 2. **Отсутствие валидации DICOM**
- Нет проверки что файлы действительно DICOM
- Нет проверки PatientName, Modality

### 3. **Проблемы с путями**
- Кириллица в путях может вызывать ошибки
- Пробелы в именах файлов

## 🔧 РЕШЕНИЯ

### ✅ Вариант 1: Использовать исправленный upload (Рекомендуется)

Я создал `backend/app/api/upload_fixed.py` который правильно обрабатывает вашу структуру:

**Что делает исправленный код:**
1. **Рекурсивный поиск** DICOM файлов во вложенных директориях
2. **Валидация** каждого файла через pydicom
3. **Организация** файлов в плоскую структуру для TotalSegmentator
4. **Поддержка** кириллицы и пробелов в путях

**Как использовать:**
```python
# Замените в main.py
from app.api import upload
# на
from app.api import upload_fixed as upload
```

### ✅ Вариант 2: Подготовить ZIP архивы (Альтернатива)

Используйте скрипт `prepare_dicom_zip.py` для создания правильных ZIP:

```bash
# Создать ZIP для всех пациентов
python prepare_dicom_zip.py "D:\DICOM СНИМКИ" "D:\DICOM_ZIP"

# Создать ZIP для конкретного пациента
python prepare_dicom_zip.py "D:\DICOM СНИМКИ" "D:\DICOM_ZIP" --patient "Мовсесян"

# Только показать список исследований
python prepare_dicom_zip.py "D:\DICOM СНИМКИ" "D:\DICOM_ZIP" --list
```

**Результат:**
```
D:\DICOM_ZIP\
├── Мовсесян_Ярика_08.09.2024_Натив_24090213.zip
├── Абрамян_...zip
└── [другие пациенты].zip
```

### ✅ Вариант 3: Ручная подготовка ZIP

1. **Создайте папку** для пациента
2. **Скопируйте** все DICOM файлы (00000001, 00000002, ...)
3. **Добавьте расширения** .dcm к файлам
4. **Создайте ZIP** из папки

## ⚠️ ВОЗМОЖНЫЕ ПРОБЛЕМЫ

### 1. **Файл слишком большой**
**Решение:** Увеличить `MAX_UPLOAD_SIZE` в config.py

### 2. **Неверный формат ZIP**
**Решение:** Проверить что ZIP не поврежден

### 3. **Нет DICOM файлов**
**Решение:** Убедиться что в ZIP есть файлы ~527KB

### 4. **Проблемы с кириллицей**
**Решение:** Использовать исправленный upload_fixed.py

## 🔄 АВТОМАТИЗАЦИЯ

### Batch загрузка всех пациентов:
```python
import os
import requests

dicom_dir = Path("D:\DICOM СНИМКИ")
output_dir = Path("D:\DICOM_ZIP")

# Создаем ZIP для всех пациентов
os.system(f'python prepare_dicom_zip.py "{dicom_dir}" "{output_dir}"')

# Загружаем все ZIP на сервер
for zip_file in output_dir.glob("*.zip"):
    with open(zip_file, 'rb') as f:
        files = {'file': f}
        response = requests.post(
            "http://5.42.97.143:8000/api/v1/upload",
            files=files
        )
    print(f"Uploaded {zip_file.name}: {response.json()}")
```

---

Эта инструкция поможет вам правильно загружать DICOM файлы с учетом вашей конкретной структуры данных.
